# 系统架构设计

## 数据流

### 主决策循环（每 60 秒）

```
时间轴 ──────────────────────────────────────────────────────▶

 ① 读取链上状态           ② 获取链下数据
 │                         │
 ▼                         ▼
 MCP Server                x402 数据源
 │ get_all_pools()         │ fetch("us_treasury_10y")
 │ get_account_balance()   │ fetch("t_bill_3m")
 │                         │
 ▼                         ▼
 ┌─────────────────────────────────────────┐
 │           ③ 构建市场快照                │
 │  pools[] + oracle_data[] + market{}     │
 └────────────────┬────────────────────────┘
                  │
                  ▼
 ┌─────────────────────────────────────────┐
 │           ④ 决策引擎                    │
 │                                         │
 │  4a. 规则预检 ─── 硬性约束过滤          │
 │      - 余额 >= min_gas_reserve?         │
 │      - APY 差异 >= 100bps?              │
 │      - 距上次 rebalance >= 300s?        │
 │                                         │
 │  4b. LLM 分析 ─── 策略建议              │
 │      - 输入：池子状态 + Oracle + 市场   │
 │      - 输出：操作类型 + 参数 + 置信度   │
 │                                         │
 │  4c. 风险校验 ─── 多维评估              │
 │      - 金额占比检查                     │
 │      - Gas 成本效益检查                 │
 │      - 波动性阈值检查                   │
 │      - 置信度下限检查                   │
 │                                         │
 │  4d. 最终决策                           │
 └────────────────┬────────────────────────┘
                  │
                  ▼
 ┌─────────────────────────────────────────┐
 │           ⑤ 执行                        │
 │                                         │
 │  if REBALANCE:                          │
 │    - 构建 rebalance 交易                │
 │    - 通过 Agent Skills 签名             │
 │    - 通过 MCP Server 提交               │
 │    - 记录到 AgentRegistry               │
 │                                         │
 │  if HOLD:                               │
 │    - 记录决策原因                       │
 │    - 无链上操作                         │
 └────────────────┬────────────────────────┘
                  │
                  ▼
 ┌─────────────────────────────────────────┐
 │           ⑥ 记录 & 通知                 │
 │  - 决策日志 → WebSocket → 前端         │
 │  - 操作结果 → AgentRegistry 合约       │
 │  - Oracle 数据 → RwaOracle 合约        │
 └─────────────────────────────────────────┘
```

## 组件交互矩阵

| 调用方 → 被调用方 | MCP Server | CSPR.cloud | x402 数据源 | YieldPool | RwaOracle | AgentRegistry |
|-------------------|:----------:|:----------:|:-----------:|:---------:|:---------:|:-------------:|
| **决策引擎** | ✓ 读取 | ✓ 历史 | ✓ 付费获取 | - | - | - |
| **执行引擎** | ✓ 提交 | - | - | ✓ rebalance | ✓ submit_data | ✓ record_op |
| **前端 API** | - | ✓ 聚合 | - | ✓ 查询 | ✓ 查询 | ✓ 查询 |
| **风险评估** | - | ✓ 波动率 | - | - | - | - |

## 安全边界

```
┌─────────── 信任边界 ────────────────────────────────────┐
│                                                         │
│  代理私钥 ─── 仅存在于代理进程内存中                    │
│              Docker volume 只读挂载                     │
│              不写入日志                                 │
│                                                         │
│  交易签名 ─── 所有链上操作必须经过风险评估              │
│              单次 rebalance 金额 <= 池总量 30%          │
│              最小 rebalance 间隔 300 秒                 │
│                                                         │
│  LLM 输出 ─── 解析为结构化 JSON                         │
│              必须通过规则校验                            │
│              不直接执行 LLM 原始文本                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 可扩展性设计

### 当前（MVP）

- 2-3 个模拟收益池
- 单个 AI 代理
- 模拟 RWA 数据源

### 中期

- 接入真实 Casper DeFi 协议
- 多代理竞争（不同策略）
- 真实 RWA 数据源（国债 API）

### 长期

- 用户跟投机制（委托资金给代理）
- 跨链 RWA 数据聚合
- DAO 治理代理参数（风险阈值、池子列表）
